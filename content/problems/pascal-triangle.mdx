## Brute Force

Idea

Use the mathematical formula:

(n/r) = n! / r!(n−r)!
	​

Compute every value independently.

Complexity

Time: O(n³) (or worse if factorials are recomputed)

Space: O(1) (excluding output)

## Better Solution

Compute each value using the previous value in the same row with combinations.

Still involves mathematical calculations and is less intuitive.

## Optimal Solution

Simulation / Dynamic Programming

## Core Intuition

Each row starts and ends with 1.

Every middle element is the sum of the two elements directly above it.

        1
      1   1
    1   2   1
  1   3   3   1
1  4   6   4   1

Formula:

current[i] = previous[i-1] + previous[i]

So instead of computing combinations, simply build each row using the previous row.

## Algorithm


Create the answer list.
Iterate from row 0 to numRows-1.
Create a new list for the current row.
For each position:
If it is the first or last element, add 1.
Otherwise, add the sum of the two numbers above it from the previous row.
Add the completed row to the answer.
Return the answer.

## Complexity

Time  O(numRows²)

Reason:
There are 1+2+3+...+n elements generated.

Total  n(n+1)/2

which is O(n²).

Space O(numRows²)

Since we store the entire triangle.

## Common Mistakes

❌ Forgetting first and last element are always 1
Wrong
Only compute using previous row

Correct
if(i==0 || i==row)
    add(1);

❌ Accessing previous row for first row
For row 0, there is no previous row.
Always handle boundary rows separately.

❌ Wrong indices
Correct:
previous.get(i-1) + previous.get(i)

Not
previous.get(i) + previous.get(i+1)

❌ Loop condition
Correct
for(int i=0;i<=row;i++)

Each row contains exactly row + 1 elements.


Why Does This Work?

Every value (except the boundaries) depends only on the two numbers directly above it.
Since the previous row is already computed, we can build the current row in one pass.
This is a classic bottom-up DP idea where previously computed states are reused.

## Interview Mantra

"Whenever each element depends only on values from the previous row, think of building the answer bottom-up. In Pascal's Triangle, the first and last elements are always 1, while every middle element is simply the sum of the two elements directly above it. This naturally leads to an O(n²) simulation (bottom-up DP) solution."

## Code (Java)

```java
class Solution {
    public List<List<Integer>> generate(int numRows) {
        List<List<Integer>> ans = new ArrayList<>();
        for(int row=0;row<numRows;row++){
            ArrayList<Integer> level = new ArrayList<>();
            for(int i = 0; i<=row; i++){
                if(i==0||i==row){ // for 1st and 2nd row
                    level.add(1);
                }else{
                    // take the sum of two element from the previous level
                    level.add(ans.get(row-1).get(i-1) + ans.get(row-1).get(i));
                }
            }
            ans.add(level);
        }
    return ans;
    }
}
```
