## Recognition

You need two values that satisfy a target relation, and indices must be distinct. Ask whether one pass can remember what has already appeared.

## Interview Trigger Words

"Find a pair", "target sum", "return indices", and an unsorted array are strong hash-map signals.

## Brute Force

Check every pair. This is simple but costs **O(n²)** time.

## Optimal Solution

Walk left to right. For each `value`, look for `target - value` in a map of earlier values. If it exists, return its index with the current index; otherwise save the current value.

```java
public int[] twoSum(int[] nums, int target) {
  Map<Integer, Integer> seen = new HashMap<>();
  for (int i = 0; i < nums.length; i++) {
    int need = target - nums[i];
    if (seen.containsKey(need)) return new int[]{seen.get(need), i};
    seen.put(nums[i], i);
  }
  return new int[0];
}
```

## Complexity

- Time: O(n)
- Space: O(n)

## Common Mistakes

Insert after checking; otherwise a value can incorrectly pair with itself.

## Interview Mantra

When a pair condition becomes a complement lookup, trade nested loops for memory.
